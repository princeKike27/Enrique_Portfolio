PROJECTS

ML: Poland House Price Calculator

June 17, 2022

For this project I analyzed data from the housing market in 3 cities in Poland: Krakow, Warsaw and Poznán. Using a dataset from Kaggle I created a Multiple Linear Regression Model that predicts a houses' price based on features like its city, the number of rooms, number of floors, its area and the year it was built. By using Gradient Descent I will find the best parameters that minimize cost and optimize the model performance.

Throughout the project you will be able to find:

  • Data cleaning and standardization.
  • Analysis of relevant features to build the model.
  • Multiple Linear Regression Model to predict a house’s price.

Datasets and Technologies Used

The analysis was done with a csv file from Kaggle. It contains the address of the house, city, number of floors, house id, latitude, longitude, number of rooms, area in square meters, year built and its price.

In terms of technologies, python was used for data analysis with the following tools: pandas for data tidying and manipulation, numpy for numerical analysis matplotlib and seaborn for plots and graphs, torch for Automatic Differentiation and Stochastic Gradient Descent for finding the gradient of cost and optimizing the model.

Data Analysis

Features for Model

The features selected for the model are: is_warsaw, number_floor, number_rooms, area_sq and is_greater_1970. In that sense, based on these characteristics a price for a house in Poland is predicted in euros. I used binning techniques on the previously mentioned features so they could be converted into numbers and be fed into the model. For example, the is_greater_1970 feature goes through every house in the dataset and maps its value to 1 if the year it was built is greater than 1970. Also, normalization was used on the area_sq feature because it is in a different scale and its distribution is skewed, after its transformation, its range is between 0 and 1. Finally, outliers and incorrect values were detected and removed from the analysis. The image below shows the features correlation heatmap.

As can be seen in the heatmap above, there is little or none Multicolinearity between the features. None of them is lower than -0.5 and only 1 is greater than 0.5. This means, that the features selected are linearly independent among them and are suitable for a Linear Regression Model.

Linear Regression Model

Continuing with what was stated previously, the data is divided into train (80% of the data) and test (remaining 20%) for the Features and Labels. With that done, forward pass is started, creating the regression function to calculate the predictions, and the parameters of the features are initialized at random values. In step 2, Cost C is calculated using Mean Square Error, which averages the quadratic cost (house_price_prediction - actual_house_price)^2 for all the houses in the training set. In step 3, C-Gradient is calculated with respect to the parameters, using Automatic Differentiation the partial derivatives of Cost C with respect to each of the parameters is found. Finally, on step 4 Gradient Descent is used in an iterative process to find the best parameters that minimize Cost C.

After 1000 epochs Cost C passed from 54.749 to 0.282. Figure 2 below shows the scatter plot of the actual house price Vs the predicted house price given by the model.
