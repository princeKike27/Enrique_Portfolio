ML: K Nearest Neighbors Book Recommendation Classifier

November 6, 2021

For this project I analyzed data from the Book-Crossings dataset which contains 1.1 million ratings, on a scale from 1-10, of 270,000 books by 90,000 users. In that sense, using the K-Nearest Neighbors algorithm I created a book classifier that based on features like the book´s year of publication and its author will predict if I will like a book, and it will recommend me other similar books to read.

Throughout the project you will be able to find:

  • Data cleaning and standardization.
  • Analysis of relevant features to build the model.
  • KNN Classifier for book prediction and recommendations.

Datasets and Technologies Used

The analysis was done with two csv files from the Book-Crossings dataset ‘Books.csv’ and ‘Book-Ratings.csv’. The former contains the isbn, title, author and year of publication of 271,379 books while the latter, has the user_id, isbn and the book rating on a scale from 1 to 10.

In terms of technologies, python was used for the data analysis with the following tools: pandas for data tyding and manipulation, numpy for numerical analysis matplotlib and seaborn for plots and graphs, sklearn.preprocessing for normalizing features and sklearn.neighbors for the K-Nearest Neighbors classifier.

Data Analysis

Features and Labels for Model

The features selected for the model are: year, is_Orwell, is_Asimov, is_Hesse, is_Doyle, is_Tolkien and is_Martin, which summarizes my favorite authors. In that sense, based on the book’s year of publication and its author the KNN classifier is going to predict if I will like the book and give me a recommendation of books I will also enjoy reading. I used binning techniques on the previously mentioned features so they could be converted into numbers and fed into the model. For example, the is_asimov feature goes through every book in the dataset and maps its value to 1 if the author of the book is Isaac Asimov, else to 0. On the other hand, I only like books that are worth reading that means, books which have an average rating greater than or equal to 7.5. With this in mind, the label worth_reading shows for each book if it is going to be worth reading. A 0 means that the book doesn´t have enough average rating for me to consider reading it, while a 1 means that based on my likes I will enjoy reading the book. K Nearest Neighbors Book Classifier

Continuing with what was stated previously, the data is divided into train (80% of the data) and test (remaining 20%) for the Features and Labels DataFrames. Before passing the train and test data to the classifier, the Features are normalized using Z-Score normalization, so that all values are in the same scale. With that done, the training data is used to create the KNN classifier, the model learns from this data and finds the best k (number of neighbors) in an iterative process. The following step, is feeding the test data, which has not been seen previously by the model, to evaluate its accuracy.

Moving forward, the goal of the K-Nearest Neighbor Book Classifier is to classify a book passed as input as worth reading and provide recommendations of similar books, that based on my likes I will enjoy reading. The algorithm uses euclidean distance to determine the distance between the book’s features. In that sense, using sklearn.neighbors the features_train and labels_train datasets are fed to the KNN model and since I only like to get 9 book recommendations k is set to 9. Book Classifier Recommendations

Taken into account what was said before, I tested the KNN classifier with two books, Tuf Voyaging a science fiction novel written in 1986 by George R.R. Martin and the Perennial Philosohpy a mysticism study written in 1945 by Aldous Huxley. I created a function get_recommendations that takes the books features as parameters, normalizes them, classifies the book as 1 if the algorithm predicts I will enjoy the book and as 0 if I won’t like it. The algorithm speaks as Multivac, in honor of Isaac Asimov’s legendary computer.
