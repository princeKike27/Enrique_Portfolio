ML: K-Vecinos Más Cercanos - Clasificador de Libros

November 6, 2021

Para este proyecto analicé los datos del Book-Crossings dataset el cual contiene, 1.1 millon de clasificaciones, en una escala de 1 a 10, de 270,000 libros por parte de 90,000 usuarios. Con esto en mente, utilizando el algoritmo K-Vecinos Más Cercano (KNN) creé un clasificador de libros que basandose en características como el año de publicación y el autor predice si me va a gustar el libro y, me va a recomendar otros libros similares para leer.

A lo largo del proyecto se encuentra:

  • Limpieza de datos y estandarización.
  • Análisis de características relevantes para construir el modelo.
  • KNN Clasificador de Libros para predicción y recomendaciones de lectura.

Archivos y Tecnologías Utilizadas

El análsis fue hecho con dos archivos .csv del Book-Crossings dataset ‘Books.csv’ y ‘Book-Ratings.csv’. El primero contiene el isbn, título, autor y año de publicación de 271,379 libros mientras que el segundo, tiene el user_id, isbn y la calificación del libro en una escala de 1 a 10.

Ahora bien, en terminos de las tecnologías utilizadas, python fue utilizado para el análisis de datos con las siguientes herramientas: pandas para limpiar, estandarizar y manipular datos, numpy para operaciones y análisis númerico, matplotlib y seaborn para gráficar, sklearn.preprocessing para la normalización de características y sklearn.neighbors para el clasificador KNN.

Análisis

Características y Etiquetas del Modelo

Las características (features) seleccionadas para el modelo son: año(year), is_Orwell, is_Asimov, is_Hesse, is_Doyle, is_Tolkien y is_Martin las cuales, resumen a mis autores favoritos. En este sentido, de acuerdo al año de publicación del libro y su autor el clasificador KNN va a predecir si me va a gustar el libro y me va a dar recomendaciones de otros libros que cree que voy a disfrutar leyendo. Debido a que el clasificador solo acepta números como input técnicas de estandarización fueron utlizadas en las características previamente mencionadas para convertirlas a números. Por ejemplo, la característica is_Asimov recorre todos los libros de la base de datos y mapea su valor a 1 si el autor del libro es Isaac Asimov, de lo contrario a 0. La tabla a continuación muestra las primeras 5 filas del Features DataFrame.

Por otro lado, solo me gustan los libros que vale la pena leer por ende, estos serían los libros que tienen una calificación promedio mayor o igual a 7.5. Con esto en mente, la etiqueta worth_reading muestra para cada libro si vale la pena leerlo. Un 0 siginifica que el libro no cuenta con una calificación promedio para que considere leerlo mientras, un 1 significa que de acuerdo a mis gustos voy a disfrutar leyendolo. La tabla a continuación muestra las primeras 5 filas del DataFrame con la columna worth_reading.

KNN Clasificador de Libros.

Continuando con lo expuesto anteriormente, los datos del Features y Labels DataFrames son divididos en entrenamiento (80% de los datos) y test (el restante 20%). Antes de pasar los datos de entrenamiento y test al clasificador, las features son normalizadas utilizando Nomalización Z-Score para que todos los valores queden con la misma escala. Una vez realizado este proceso, los datos de entrenamiento son utilizados para crear el clasificador KNN, el modelo aprende de estos datos y encuentra la mejor k (número de vecinos) en un proceso iterativo. El siguiente paso es, alimentar el modelo con los datos de test, que no han sido expuestos previamente al modelo, para evaluar su precisión.

Con esto en mente, el objetivo del clasificador KNN es clasificar los libros que se pasan como input como merecedores de ser leidos y dar recomendaciones de libros similares, que de acuerdo a mis gustos, voy a disfrutar leyendo. El algoritmo utiliza la distancia euclidiana para determinar la distancia entre las diferentes características de los libros. Así pues, utilizando sklearn.neighbors los datasets features_train y labels_train son alimentados al clasificador KNN y, debido a que solo quiero 9 recomendaciones k es igual a 9. Recomendaciones del Clasificador KNN

Teniendo en cuenta lo expuesto previamente, probe el clasificador con dos libros, Los Viajes de Tuf una novela de ciencia ficción escrita por Geroge R.R. Martin en 1986 y La Filosofía Perenne un estudio místico escrito por Aldous Huxley en 1945. Para esto, creé una función get_recommendations que toma como parametros las características de los dos libros, las normaliza, clasifica cada libro como 1 si el algoritmo predice que me va a gustar el libro y como 0 si no me va a gustar. El algoritmo se hace pasar por Multivac, en honor al computador legendario de las hsitorias de Isaac Asimov. Los resultados de la prueba fueron que ambos libros se clasificaron como 1, una impresión de los resultados de la función con las recomendaciones y autores para cada libro se pueden ver a continuación.
